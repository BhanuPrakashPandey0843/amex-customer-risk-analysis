# docs/

## Purpose
This directory contains project documentation about the analysis: the assignment brief, problem statement, methodology, audit, and future plans.

## File Listing

| File | Description |
|------|-------------|
| `assignment_brief.md` | **Source document**: The original assignment brief verbatim, preserved for auditability |
| `problem_statement.md` | Formal problem statement and scope definition |
| `business_question.md` | The 10 specific business questions this analysis answers, with references |
| `methodology.md` | End-to-end methodology and design principles |
| `project_audit.md` | Self-audit against the assignment's evaluation criteria |
| `project_dependency_map.md` | Full file dependency graph of the project |
| `CHANGELOG.md` | Project change history |
| `power_bi_dashboard_plan.md` | Specification for the Power BI dashboard (future work) |

## Key Concepts

### Business Questions
The analysis is structured to answer 10 specific business questions (see `business_question.md`), prioritized by importance to senior leadership.

### Methodology
The analysis follows a 10-phase consulting workflow:
1. Project Foundation
2. Data Audit
3. Data Cleaning
4. EDA
5. Advanced Analytics
6. Risk Framework
7. Insights & Recommendations
8. Executive Storytelling
9. Power BI Dashboard (future)
10. Final Review & Validation

See `methodology.md` for full details.

### Evaluation Criteria
The self-audit in `project_audit.md` scores the project against the assignment's criteria:
- Analytical approach and problem framing
- Quality and relevance of insights
- Business recommendations
- Risk identification and mitigation
- Data storytelling and presentation quality
- Clarity, structure, and conciseness

## How to Navigate
1. Start with `assignment_brief.md` to understand the requirements
2. Read `problem_statement.md` and `business_question.md` to see how we framed the problem
3. Review `methodology.md` for how we executed it
4. Check `project_audit.md` for the self-assessment
