# notebooks/

## Purpose
This directory contains Jupyter Notebooks that document the methodology and reasoning behind each phase of the analysis.

## Important: Not Execution Artifacts
**The notebooks in this directory are documentation, not the pipeline.
- They show the thinking process step-by-step for auditability
- The actual deliverables are generated by `src/run_pipeline.py`, not by running these notebooks
- This is an intentional design choice to avoid "notebook drift" where a notebook is edited after running and the outputs no longer match the code

## Directory Listing

| Notebook | Phase | Purpose |
|----------|-------|---------|
| `01_data_audit.ipynb` | Phase 2 (Data Audit) | Documents the initial data profiling, quality assessment, and issue identification |
| `02_data_cleaning.ipynb` | Phase 3 (Data Cleaning) | Documents the cleaning logic, feature engineering, and validation steps |
| `03_eda.ipynb` | Phase 4 (EDA) | Documents the exploratory analysis, visualizations, and initial findings |

## How to Use
1. **Read-only**: These notebooks are for understanding the methodology
2. **Reference**: Use them to see how decisions were made about cleaning, feature engineering, etc.
3. **Reproducibility**: The actual pipeline code is in `src/`, but these notebooks show how it was developed

## Note on Deprecated Notebooks
Older draft notebooks are in `_DELETE_ME/` for reference only. They are not part of the final deliverable.
