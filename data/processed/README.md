# data/processed/

## Purpose
This directory contains the **analysis-ready cleaned dataset** and **computed metrics** that power all reports, charts, Excel, and PDF outputs.

## Files

### Core Analysis File
| File | Description |
|------|-------------|
| `complaints_cleaned.csv` | The primary cleaned dataset used for all analysis — 196,835 rows, 22 columns (13 original + 9 engineered) |

### Single Source of Truth
| File | Description |
|------|-------------|
| `eda_metrics.json` | **Critical file**: Every number in every report, chart, Excel, and PDF comes from this JSON file. It is generated by `src/metrics_engine.py` and is the single source of truth. No downstream artifact recomputes anything independently. |

### Audit & Quality Files
| File | Description |
|------|-------------|
| `audit_summary.csv` | Summary statistics from the initial data audit phase |
| `metadata_summary.csv` | Metadata about the dataset (column types, counts, etc.) |
| `missing_values_analysis.csv` | Breakdown of missing values by column pre-cleaning |
| `quality_scores.csv` | Data quality scores pre- and post-cleaning (7.5 → 8.5/10) |
| `cleaning_log.csv` | Full audit trail of all transformations applied during data cleaning |

## complaints_cleaned.csv Schema
The cleaned dataset has 22 columns:
1. Original 13 columns from `raw/` (cleaned and standardized)
2. 9 engineered features:
   - `complaint_year` (integer)
   - `complaint_quarter` (1–4)
   - `complaint_month` (1–12)
   - `date_received_dt` (UTC-aware datetime)
   - `company_clean` (standardized company name)
   - Plus 4 additional analytical features

Full data dictionary: [`reports/data_dictionary.md`](../../reports/data_dictionary.md)

## Data Quality (Cleaned)
- **Overall Score**: 8.5/10 (+1.0 improvement)
- **Completeness**: 100% of records preserved
- **Validity**: Dates parsed, names standardized
- **Consistency**: Formatted consistently across all rows
- Full details: [`reports/data_quality_comparison.md`](../../reports/data_quality_comparison.md)

## Regeneration
All files in this directory can be regenerated by running:
```bash
python src/run_pipeline.py
```

## Important Notes
- `eda_metrics.json` should never be edited manually. If you need to change a metric, modify `src/metrics_engine.py` and re-run the pipeline.
- `complaints_cleaned.csv` is generated by `src/data_cleaning.py`
